# SLO Definitions and Dashboards

## Service-level objectives

| SLO | Target | Window |
|-----|--------|--------|
| Run availability | 99.9% | 30 days |
| Run P99 latency | 500 ms | 1 hour |
| Journal append P99 | 100 ms | 1 hour |

## Error budget

The error budget for the 99.9% availability SLO allows 0.1% of requests to fail over 30 days.

The budget remaining fraction is computed as:

```
budget_remaining = 1 - (observed_error_rate / allowed_error_rate)
```

When `budget_remaining` drops below 0.0, the SLO is breached.

## Burn-rate alerts

Burn-rate alerts fire when the error budget is being consumed faster than the sustainable rate.

| Alert | Burn Rate | Meaning |
|-------|-----------|---------|
| Fast burn | > 14.4x | Budget exhausted in 2 hours |
| Slow burn | > 3x | Budget exhausted in 10 days |

Configure with `BurnRateAlert::new(threshold_multiplier)`.

## Grafana dashboard

The dashboard is generated by `grafana_dashboard()` and exports JSON compatible with Grafana 9+.

Panels:
- Run success rate (timeseries)
- Run latency P99 (timeseries)
- Error budget remaining (gauge)
- Worker utilization (timeseries)
- Queue depth (timeseries)

Variable: `$tenant` (auto-populated from metric labels).

## Importing the dashboard

```bash
curl -X POST http://grafana:3000/api/dashboards/import \
  -H 'Content-Type: application/json' \
  -d "{\"dashboard\": $(cargo run --bin export-dashboard), \"overwrite\": true}"
```
